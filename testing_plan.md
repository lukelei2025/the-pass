# 测试计划 - The Pass (Manual QA)

为了确保全栈集成和品牌重塑后的稳定性，请按照以下步骤进行系统性测试。

## 1. PWA 安装与启动
- [ ] **桌面端测试**：在 Chrome/Edge 浏览器地址栏右侧寻找“安装”图标，确认名称为 **The Pass**。
- [ ] **移动端测试 (iOS)**：在 Safari 点击“分享” -> “添加到主屏幕”，确认图标是否显示正常且无白边。
- [ ] **独立窗口**：启动安装后的 App，测试是否没有浏览器地址栏，具有原生 App 感。

## 2. 核心工作流 (The Pass / Mise-en-place)
- [ ] **添加链接**：粘贴一个飞书或微信链接，验证 **LLM 自动分类** 是否生效，且 **Title/Note** 是否正确分离。
- [ ] **添加笔记**：直接输入文字，确认是否存入 Mise-en-place。
- [ ] **手机端 FAB**：在手机端点击右下角蓝色 “+” 按钮，确认是否平滑滚动并聚焦输入框。

## 3. 页面流转 (术语一致性)
- [ ] **Mise-en-place -> Le Service**：点击卡片（手机）或悬浮点击 **Tick** 按钮，确认项目移入 **Le Service** 页面。
- [ ] **Mise-en-place -> La Réserve**：点击卡片（手机）或悬浮点击 **Stow** 按钮，确认项目移入 **La Réserve** 页面（带重音标题）。
- [ ] **查看记录**：点击卡片或悬浮点击 **Fini**，确认项目出现在 **Le Journal** 中，且状态显示为 "Fini"。

## 4. 移动端专项 UX
- [ ] **底栏菜单 (Action Drawer)**：在手机端点击任何卡片，底栏是否从下方滑出。
- [ ] **遮罩关闭**：点击底栏菜单以外的阴影区域，菜单是否正常关闭。
- [ ] **二次确认**：点击 **Void** (删除) 时，是否有浏览器原生确认弹窗。

## 5. 多端同步
- [ ] **实时性**：在电脑上添加一个项目，确认手机端是否在 1-2 秒内自动同步显示（无需手动刷新）。
- [ ] **登录状态**：退出并重新登录，确认历史数据（Le Journal）和已存储项目（La Réserve）是否完整加载。
